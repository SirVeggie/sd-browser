IMG modes

Mode keyword first, then space-separated hex ids (no + required). Weighted +/- stays for mixed text and subtract. Fringe is in.

Syntax

IMG <64hex> [threshold] [k]                 # single image
IMG turtle …                                # text
IMG turtle + <hex> - beach                  # weighted mix (unchanged)

IMG avg    <id> <id>… [threshold] [k]       # centroid
IMG all    <id> <id>… [threshold] [k]       # match all (geom mean)
IMG any    <id> <id>… [threshold] [k]       # match any (max)
IMG more   <idA> <idB> [threshold] [k]      # extrapolate A past B
IMG fringe <id> <id>… [threshold] [k]       # related but atypical
Parse rule: treat the first token as a mode only when it is a known mode name and it is followed by at least one 64-char hex id. Otherwise keep today’s text/weighted path so IMG fringe of trees still works as text.

IMG any id1 id2

Math: max similarity to any ref

Gets you: moodboard / multi-theme neighbors

fast KNN

more

IMG more idA idB

Math: normalize(A + α(A−B)), α≈1

Gets you: exaggerate what distinguishes A from B

score-space

fringe

IMG fringe id1 id2

Math: union neighborhood; high near-ref sim, low centroid sim

Gets you: weird cousins, hybrids, near-misses

What stays as-is

  • Single image: IMG <hex> 0.8 from Similar images
  • Weighted mix: IMG a + b - beach — soft AND + subtract, including hex ids with +
  • SIMILAR: prompt-only
