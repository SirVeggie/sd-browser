Multi-image similarity search

Ways to turn a selection of images into a search — beyond averaging embeddings — and what each mode tends to return.

Baseline Consensus Coverage Contrast Set / novelty Compare When to use what Shortlist

Baseline

Today: one reference via IMG <64-char-id> → cosine KNN. Multiple SIMILAR clauses merge by min-score intersection. Weighted text IMG already does geometric mean of positives − max of negatives. Selection exists; no multi-reference similar action yet.

Centroid / average (the obvious first mode): mean of reference vectors, L2-renormalize, then KNN. Soft consensus — shared subject/palette/composition. Weak refs get washed out; conflicting styles collapse into a muddy middle.

Fast KNN — single query vector Score-space — scan / merge neighborhoods Good shortlist candidate

1. Consensus — shared traits

A
shortlist fast KNN

Centroid (average)

Images near the common center of the selection.

Query mean of vectors, renormalize

Feels like soft consensus / blend

Fails when refs disagree hard (portrait + landscape) — bland midpoints that match neither.

B
shortlist score-space

Soft intersection

Must resemble every reference at least somewhat.

Score min(simi) or geometric mean — same idea as weighted IMG positives

Feels like “could sit next to all of these,” not “looks like the average”

Stricter than centroid; empties out faster when the set is mixed.

C
score-space

Hard intersection

Pass a threshold against each ref; rank by min or mean.

Feels like tight theme clusters, or nothing

Note closest to today’s multi-SIMILAR merge

D
fast KNN

Median / geometric median

Like centroid, but one odd pick doesn’t drag the query as hard.

Best when set is mostly cohesive with 1–2 outliers

2. Coverage — “any of these”

E
score-space

Soft union (max)

Neighbors of any reference — N similar searches merged by best score.

Feels like broad, multi-modal gallery

Strongest ref often dominates the ranking unless you diversify.

F
shortlist score-space

Union + MMR

Same pool as union, then MMR so results don’t collapse onto one ref.

Feels like moodboard expansion — coverage across modes

Reuse existing MMR patterns

G
score-space

k-medoids / exemplar cover

One pile of neighbors per exemplar, interleaved.

Feels like clearer multi-theme than raw max-score when refs are far apart

3. Contrast, difference, direction

H
shortlist needs +/−

Positive − negative

“Like these, but not like those.”

Score / query reuse weighted IMG formula, or normalize(mean(pos) − λ·mean(neg))

UX include/exclude groups, or last-clicked as negative

Classic win: keep subject, drop style — or the reverse.

I
fast KNN

Difference / residual

Images along the axis of change between refs, not “similar to A”.

Query normalize(A − B)

Noisy as a pure KNN query; often better as a re-ranker on a broader pool.

J
shortlist fast KNN

Extrapolation

More of whatever distinguishes A from B.

Query normalize(A + α(A − B))

Feels like generative / exaggerated retrieval

Surprising at high α; unstable when A ≈ B.

K
fast KNN

Shared subspace

Drop dimensions that disagree across the selection; query on what stays stable.

Cost tiny PCA/SVD on N≤20 refs — cheap, a bit more engineering

4. Set structure and novelty

L
score-space

Analogy (A:B :: C:?)

Same relationship in embedding space — relational transfer.

Feels like the most novel mode; needs the embedding to actually encode that axis

M
score-space

Set affinity

Does this candidate belong with the group, not just near the centroid?

Feels like curation / “complete this collection”

Quirk can favor bridges between two clusters

N / O
shortlist score-space

Fringe / anti-centroid

Related to the selection, but atypical — edge cases, hybrids, weird cousins.

Score high max-sim to nearest ref, low to centroid; or uniqueness within the union neighborhood

Pairs with MMR after a loose union

P
score-space

Cluster-conditioned

If the selection has multiple themes, search per cluster and show split/interleaved results.

UX “your selection has 2 themes” — honest multi-theme instead of one muddy average

When selection shape changes the answer

Refs cohesive?

Yes → consensus

Centroid or soft intersection. Modes barely differ when the two refs are already similar.

Refs far apart?

One theme vs many

Want one theme → intersection (may empty). Want many → union + MMR or medoids. Centroid muddies.

Have excludes?

Pos − neg

Keep subject / drop style (or reverse). You already have this score shape for text IMG.

Want novelty?

Fringe or extrapolate

Fringe for weird cousins. Extrapolation when you have a clear A-vs-B direction to push.

  • Many near-duplicates: median or soft intersection beats plain mean.
  • Style refs mixed with content refs: pos/neg or shared subspace beats averaging.
  • Implementation note: prefer stored embeddings for selected ids — re-embedding N refs via API each search will hurt.

Recommended shortlist

Six modes that feel distinct in a UI without overlapping too much.

1
Average (centroid) Default. Familiar soft consensus.
2
Match all (soft intersection) Stricter theme — must fit every pick.
3
Match any (union + light MMR) Multi-theme moodboard expansion.
4
Like these / not those Reuse weighted pos/neg for controlled subtract.
5
More like A than B (extrapolation) Novelty / exaggeration for 2+ images.
6
Fringe of selection Discovery — related but atypical.
Design catalog only — not an implementation plan. Fast path = single vector into existing sqlite-vec KNN; rich path = score-space scan like weighted IMG.
