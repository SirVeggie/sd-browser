Beyond the v1 shortlist

Four modes that dig deeper

Difference, shared subspace, analogy, and set affinity — what they actually optimize for, how they differ from avg / more / all, and when they’d be worth building.

Difference Shared subspace Analogy Set affinity

Difference / residual

single query vector deferred

Search along the axis of change between two images — not “near A,” not “blend of A and B.”

Intuition

Pick a watercolor portrait (A) and a photo portrait (B). Averaging pulls toward “generic portrait.” Extrapolation (more) says “even more watercolor than A, away from B.” Difference throws away absolute position and keeps only the direction photo → watercolor (or A−B). You’re asking: “who else in the library sits on that same transformation axis?”

Math

q = normalize(A − B) score(c) = cosine(c, q)

Optionally recenter: score candidates that are also somewhat near A, or re-rank a pool from any A B by alignment with A−B. Pure difference KNN with no anchor is the noisiest form.

What shows up

  • Style transfers that share the same “delta” (photo→paint, day→night, clothed→outfit-X)
  • Images that aren’t near A or B in absolute terms, but move in a similar direction from some baseline
  • Often abstract / surprising — can feel magical or like garbage depending on the embedding
vs more (extrapolation) more stays near A and pushes past it. Difference can leave A’s neighborhood entirely and chase the direction alone.
vs pos − neg Weighted subtract still rewards being like positives. Difference doesn’t require looking like A — only aligning with A−B.

Fits IMG as

IMG diff <idA> <idB>

Exactly two ids. Order matters (A−B ≠ B−A).

Cosine to a difference vector is brittle when A≈B (tiny noisy direction) or when the embedding doesn’t linearly encode the trait you care about. Best as a re-ranker on a broader pool, or with a soft anchor to A.

Shared subspace

single query vector

Ignore what the selected images disagree on; search using only what they stably share.

Intuition

You select five images of the same character in different outfits, lighting, and crops. Centroid still averages the outfits in. Soft all demands every outfit trait a bit. Shared downweights embedding dims that vary within the set — what’s left is closer to “character identity” (or shared medium, shared composition — whatever is stable).

Math

mean = average of ref embeddings var_i = within-set variance of dim i ε = median(var) // relative to this set; not absolute 1 q_i = mean_i / (var_i + ε) q = normalize(q) then ordinary KNN

Absolute mean/(1+var) is a no-op on L2-normalized high-d embeddings (var ≪ 1). Median-scaled inverse-variance actually reweights peer dims. Cheaper than PCA; enough for small N.

What shows up

  • More of the same subject / motif despite style churn in the selection
  • Fewer “outfit average” or “lighting average” false friends than plain avg when those traits still pollute the mean
  • If the set only shares something weak (all “colorful”), you get weak/generic results
vs avg Avg keeps disagreeing traits at whatever strength they survive averaging. Shared crushes high-variance dims relative to stable ones.
vs all all requires candidates to match each ref’s private traits too. Shared forgives private traits.

Fits IMG as

IMG shared <id> <id>…

Needs ≥2 ids. With nearly identical refs, collapses to avg (all vars ≈ 0).

Assumes “disagreement within the set = nuisance.” Wrong when you wanted the varying trait (five different art styles and you care about style range). Axis-aligned only — correlated multi-dim variation is only partially suppressed.

Analogy (A:B :: C:?)

score-space or pair search deferred · richest / hardest

Transfer a relationship. “A is to B as C is to what?” in embedding space.

Intuition

Classic word2vec energy: king − man + woman ≈ queen. Visually: sketch-of-cat − photo-of-cat + photo-of-dog ≈ sketch-of-dog — if your embedding encodes style and subject somewhat linearly and separately. You’re not searching “similar to C”; you’re applying the A→B transform at C.

Math

// three-ref form (most natural UX) q = normalize(C + (B − A)) score(d) = cosine(d, q) // four-image / pair form (harder UX) find d maximizing cosine(d − C, B − A) or cosine(normalize(d−C), normalize(B−A))

Three-id form is basically anchored difference: difference direction B−A, applied at C. That’s why it sits next to diff / more in the design space — same linear algebra family, different intent.

What shows up

  • Same subject, transferred style/medium/pose — when it works
  • Plausible near-misses that reveal what the model actually factors
  • Nonsense when the relationship isn’t linear in the embedding (often)
vs more more A B = push from B through A. Analogy needs a third point C as the place to apply A→B.
vs diff Diff searches the direction alone. Analogy searches “C plus that direction.”

Fits IMG as

IMG analogy <A> <B> <C>

Read as “A is to B as C is to ?”. Order is sacred. Pair-picking UX is awkward in multi-select unless you define order as selection order or first/last.

Highest coolness / lowest reliability of the four. Worth a prototype once more and diff exist — same vector ops, third id. Don’t bet the default UX on it.

Set affinity

score-space

Not “near the average” — “would this image belong in the set?”

Intuition

Think curation. Your selection is a little collection. Some candidates sit on the centroid but would look weird in the group (wrong vibe). Others sit off-centroid but act as a bridge between two clusters in the selection and make the set feel more complete. Affinity scores how well the candidate fits the structure of the set.

Math

// consistency-penalized membership μ = mean_i sim(c, ref_i) σ = stdev_i sim(c, ref_i) score(c) = clamp(μ / (1 + λσ)) // λ = 1 // Rejected: cohesion gain ranks identically to avg // (ref pairwise baseline is constant → only Σ sim(c, ref) varies)

Mean similarity alone is avg in ranking disguise. The σ penalty is what earns the name: one-ref specialists (high max, uneven profile) drop; even membership across the set rises. Softer than all (geom mean zeros on a single miss) and tighter than any.

What shows up

  • Collection completers: images that “fit the moodboard”
  • Fewer lonely outliers than any; less muddy than avg on mixed sets
  • Graded scores when a set has legitimate diversity (unlike hard geom-mean all)
vs all all = geom mean (harsh on a single weak link). Affinity stays graded via μ/(1+σ).
vs avg Same mean signal, but uneven match is penalized — rankings diverge when a high-μ specialist competes with a lower-μ even fit.
vs fringe Fringe wants atypical members of the neighborhood. Affinity wants typical-enough members of the set — almost opposites.

Fits IMG as

IMG affinity <id> <id>…

Natural multi-select mode for “expand this collection.” Score-space scan (not KNN); default threshold matches other image-similarity modes.

Still O(N refs × candidates). Fine for small reference strips; for huge galleries prefer a neighbor shortlist then affinity re-rank if needed.
Closest cousins: more ≈ anchored push; all ≈ strict soft-AND; fringe ≈ opposite of affinity. Difference and analogy are one linear-algebra family; subspace is “smarter avg”; affinity is consistency-penalized membership.
